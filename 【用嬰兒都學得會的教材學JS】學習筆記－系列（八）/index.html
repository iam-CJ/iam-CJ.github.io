【用嬰兒都學得會的教材學JS】學習筆記－系列（八）

發表於 2023-05-20 更新於 2023-05-30 分類於 JavaScript

前言小廢話

這系列就是讓自己做各種小專案啦～只給一些關鍵字以及作業要求，其他像是邏輯、功能和結構等等，通通由自己一人構思、完成，覺得還滿興奮的，可以測試看看自己的能力，同時也能累積投遞履歷用的作品集(`･∀･)b

課程開始

第一課－碼表應用程式

  • 課程概要：
    1. 目標：做出碼表應用程式
    2. 關鍵字：online stopwatch、線上碼表
  • 作業心得：做這個作業滿卡的，因為我想要將setInterval放入if裡面，不管怎麼宣告setInterval都一直無法使用clearInterval將計時器暫停，看到某位同學的作業是將setInterval賦值給 data 內的參數才終於恍然大悟 XD，最後成功了，然後我樣式真的沒辦法弄漂亮 ∠( ᐛ 」∠)＿

    作業說明：做出簡易的計時碼表，需要有開始按鈕、重設按鈕，按下開始按鈕後碼表開始計時，並且開始的字樣變成暫停，再按下暫停按鈕，字樣變成繼續，而按下重設按鈕碼表要歸零。
    我使用到的關鍵語法：setInterval、cleanInterval。
    https://jsfiddle.net/iamCJ/2hLfa0tw/

第二課－筆記應用程式

  • 課程概要：
    1. 目標：做出筆記應用程式
    2. 關鍵字：write notes online、online note tool
  • 作業心得：一開始做得很順，但是做到「點擊其他的事項進行修改」的部分就卡住了，將事項的內容傳送回textarea後就不知道該怎麼把修改後的內容傳回存放事項的陣列，後來看了其他同學怎麼處理的，發現有位同學是連textarea也做 for 迴圈，每個事項都有對應的textarea，而要不要顯示textarea的狀態存在事項的物件中，這樣就能修改個別的事項內容了，這個做法滿聰明的餒(`･∀･)b

    作業說明：做出簡易的記事本，有一個可以輸入文字的textarea還有新增筆記的按鈕，筆記內容要隨時存到 local storage，還需要有個筆記清單的區塊來放置所有事項，每個事項旁都有刪除按鈕，且點擊單一事項可以修改該事項的內容
    https://jsfiddle.net/iamCJ/21j9qdv8/

第三課－分帳應用程式

  • 課程概要：
    1. 目標：做出分帳應用程式
    2. 關鍵字：分帳應用程式、分帳 app、splitwise 網頁版
  • 作業心得：前陣子跟朋友們討論出遊相關事宜時，才認識到這個酷炫的新東西，沒想到我也可以做得出來！雖然我覺得思考邏輯以及安排好資料格式有點燒腦，某個地方卡住了就會想很久，或是發現取資料時很麻煩、甚至取不到，就要再改寫資料的格式，而且功能沒有像別人做的那麼複雜，畫面也沒那麼精美 ∠( ᐛ 」∠)＿

    作業說明：做出簡易的分帳程式
    作業要求如下：（1）資料要隨時存入 local storage（2）需要有可以增加成員姓名、增加款項的按鈕，而增加款項的部分可以填入金額、事項以及由誰付款（3）需要成員列表及款項列表將所有成員及所有款項列出，成員列表的部分需列出成員姓名以及應付/應收金額（並需有顏色區別），款項列表則需列出金額、事項以及付款人，並且有刪除按鈕可以刪除個別款項（4）需要有旅程結束的按鈕，清除所有成員及款項資料
    https://jsfiddle.net/iamCJ/xgu5k1vy/

第四課－行事曆應用程式

  • 課程概要：
    1. 目標：做出行事曆應用程式
    2. 關鍵字：calendar app ui
    3. 補充：現在大部份新出的 JS 套件，都會提供 native（原生 JS）、Vue、React 三種版本讓使用者做選擇、以便使用，套件官方通常會將框架版本的底層寫法做修改，不過，沒有的話也能自行將初始化相關設定放在 mounted 裡，在狀態更新時，想辦法與套件狀態同步即可
  • 作業心得：將原生 JS 環境的套件改寫成適用在 Vue 環境的過程還滿有趣的，不斷推測、嘗試可行的方法，會一點一點找到能執行的規則，不過可以的話，以後還是盡量找支援 Vue 框架的套件就好了 ∠( ᐛ 」∠)＿

    作業說明：做出簡易的行事曆，需要有一個新增活動的按鈕，點擊按鈕後可以輸入活動名稱、選擇活動日期與時間，選擇日期時間的 UI 用之前課程教過的 datetimepicker 套件，而行事曆 UI 的部分則使用 FullCalendar 套件，需要將活動正確顯示於行事曆上，並且點擊活動可以出現確認刪除的對話方塊，確認後能刪除活動，並且使用 CDN 版本，自己學習如何在框架內導入套件，不能用官方提供的 Vue 版本
    https://jsfiddle.net/iamCJ/cxrqgt0j/

第五課－體重追蹤應用程式

  • 課程概要：
    1. 目標：做出體重追蹤應用程式
    2. 關鍵字：weight tracking
    3. 補充：找尋套件但官方並無提供框架版本的話，可以搜尋how to use 套件名稱 in 框架名稱看看有無其他開發者將該套件改寫成框架可使用的版本
  • 作業心得：沒想到 Chart.js 跟 Vue 相容性有點差，總是會遇到很多未知的錯誤，不過也還好不是只有我一個人遇到這些問題，參考一下他人的解決辦法就順利完成了！

    作業說明：做出簡易的體重追蹤工具，需要有可以輸入體重及日期的表格，以及新增資料的按鈕，並且有清單列表將所有資料依日期排序列出，搭配 Chart.js 將資料圖表化（以折線圖的形式呈現）
    https://jsfiddle.net/iamCJ/tncuzgd8/

第六課－天氣查詢應用程式

  • 課程概要：
    1. 目標：做出天氣查詢應用程式
    2. 關鍵字：weather app ui
    3. 補充：關於 API 的使用，為了避免 API 被過度呼叫、濫用，通常會需要註冊、取得金鑰（key）才能呼叫，而使用金鑰驗證的方法有很多，可能是呼叫 API 時，在 http request header 裡面附上 API key，或直接把 key 附在呼叫的 url 上，也或者是在 http post 內使用 form data 附上，而實際驗證的方法還是要看 API 作者是選擇哪一種
  • 作業心得：覺得呼叫 API 還滿好玩的，不複雜且可以運用的資料很多種，加上氣象局把資料整理得很清楚，所以在學習使用的時候還滿輕鬆的，但是我偷懶，作業呈現的資料跟樣式沒有另外做處理 ∠( ᐛ 」∠)＿這次的練習想要把重點放在 call API，並取到自己想要的資料

    作業說明：做出簡易的天氣查詢工具，需要有選單選取縣市，選取縣市後需要呈現該地的天氣現象、最高溫度、最低溫度、降雨機率，而資料取用自中央氣象局的 API
    https://jsfiddle.net/iamCJ/9odgx75c/

第七課－國家資訊與地圖應用程式

  • 提醒：本次作業用到的 restcountries API，不時可能因點擊數太多、伺服器負荷量爆炸，抑或作者暫時關閉 API，導致呼叫 API 失敗而無法正常顯示
  • 課程概要：
    1. 目標：組合多個 API，做出國家資訊與地圖應用程式
    2. 使用 API 與套件：restcountries、Leaflet、OpenStreetMap
  • 作業心得：之前自己做違規照相桿地圖時，就是用 Leaflet 搭配 OpenStreetMap，所以用起來很快就上手，遇到比較多問題是 call restcountries 資料，有時會取不到想要的，有時在 function 讀不到，後來參考其他同學的結構才順利解決，但還是不太明白我原本的寫法為何出錯就是了 ∠( ᐛ 」∠)＿

    作業說明：做出簡易的國家資訊與地圖工具，顯示世界地圖，在每個國家的中心點放置大頭針的標記，點擊大頭針後會跳出視窗，視窗內顯示國旗圖案、國家名稱、首都名稱、貨幣符號
    https://jsfiddle.net/iamCJ/m30ayj8g/

小結

要做這麼多小專案真的有難度耶，每個都需要耗時三到四天左右，加上有幾課要求套用原生 JS 版本的套件，處理兼容和 bug 就佔了很多時間，不過看到專案完成且能成功執行還是非常有成就感！也對於自己的能力更有自信了一點，期望自己能製作出敢分享給身邊親朋好友使用的工具 XD
