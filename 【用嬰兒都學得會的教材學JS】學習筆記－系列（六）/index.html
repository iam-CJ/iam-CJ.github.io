【用嬰兒都學得會的教材學JS】學習筆記－系列（六）

發表於 2023-05-15 分類於 JavaScript

前言小廢話

這個系列比較像是過去的土法煉鋼寫法與接觸新框架中間的銜接（？，站長大大選擇先用 data model 與 render function 教我們關注點分離的概念，習慣資料與畫面、功能分開管理，了解於原理後再讓我們接觸框架，而非直接進入框架的教學，比較能避免我們會出現搞不清楚某些功能到底是原生 JS 還是框架所提供的狀況。

課程開始

第一課－認識 data model 與 render function

  • 課程概要：
    1. 認識 data model 的觀念：當你要面對的是大型專案時，不建議再使用操作 DOM 元素去更新其內容，會變得很難以維護，因此需要將程式的狀態與其他部份分開，而這個狀態通常被稱為 state、model 或 data model，在資料更動時，只需要去處理 data model，這樣在開發複雜專案時，相關函式一律只要專注於 data model 就好，不用管使用者介面
    2. 認識 render function 的觀念：狀態由 data model 管理，而負責更新畫面則可以使用 render function 的方法，放一個 root 元素，作為放關於使用者介面之程式碼的容器，接著寫一個 render 函式，將所設計的使用者介面做出來、塞到 root 中
      這個 render 函式有三個注意事項：
      1. 第一行要先清空放使用者介面的容器
      2. 在所有跟狀態有關操作的最後一行，都要呼叫 render 函式（為了更新畫面）
      3. 所有 DOM 操作一律由 render 函式處理
  • 作業心得：我自己覺得這堂課的觀念有點像 Vue 框架分開管理資料以及模板（或者說 HTML 要長的樣子）的概念

    作業說明：使用 render function 把 data model 給渲染出來
    https://jsfiddle.net/iamCJ/53r4on0a/

第二課－認識陣列操作

  • 課程概要：學會使用.push()新增值到陣列，括號內填入想要加入陣列的字串、數字或參數等等，它會加在陣列的最後方
  • 自己補充：其他常見的陣列處理方法
    1. .unshift()：也是新增值到陣列，但是它會加在陣列的最前方
    2. .shift()：移除陣列的第一個值
    3. .pop()：移除陣列的最後一個值
    4. .splice(開始索引, 刪除數量, 要插入的值)：用來切割移除或拼接某個陣列，這個方法會改變原陣列的值！
      括號內第一個參數填入開始的索引值，負數則代表從後面開始算起（例如 -1 為倒數第一個）；第二個參數填入從開始索引（包含）算起總共要刪除的數量，填 0 就不會刪除；第三個參數可以填入要插入的值，會插在被切割掉的值之位置
      而直接呼叫.splice()則會回傳被刪除的值
    5. .slice(開始索引, 結束索引)：用來複製某個陣列，會回傳指定索引範圍的新陣列，這個方法不會改變原陣列的值。
      括號內填索引範圍，前面的參數為開始的索引值，負數代表從後面開始算起；而後面的參數為結束的索引值，需要注意範圍會切在這個索引之前，不包含結束索引的值！沒有寫則預設為array.length
    6. .concat()：用來合併多個陣列，不會改變原先陣列的值，而是回傳一個新的、被合併的陣列。p.s.而當括號內不填入其他陣列時，我自己會當作是用來複製陣列
    7. .include()：用來尋找陣列中是否包含某個值，結果會以布林值型態回傳
    8. .indexOf()：用來尋找陣列中某個值的索引位置，而若搜索的值不在陣列內，則會回傳-1
    9. .join()：用來將陣列中的值轉為字串，並且加入指定字元做為值與值連接的符號
  • 作業心得：作業本身很簡單，不過自己補充的部分花了一些時間整理，但果然還是要自己實際筆記起來，才會記得更熟、把細節搞清楚呀 ∠( ᐛ 」∠)＿

    作業說明：延續上一堂作業，加入一個 input 以及一個新增按鈕，按鈕按下後，必須先更新 data modal 再執行畫面的 render
    https://jsfiddle.net/iamCJ/L5vrodn7/

第三課－認識匿名函式

  • 課程概要：
    1. 學會使用.splice()，根據索引刪除陣列元素：這個在上一堂我就先自己補充啦～可以點這邊來看
    2. 認識for in迴圈：參數會回傳陣列的索引，而for of則是回傳陣列的值
    3. 認識匿名函式（anonymous function）：動態宣告一個函式，然後直接指派、使用，其實就只是沒有名字的函式而已。
  • 作業心得：忘記畫面是根據 data 渲染的，資料變化就會自動更新，原本還想說處理完陣列內的資料，接著要處理<li>、<span>，結果點預覽畫面測試的時候才發現不需要多此一舉 ∠( ᐛ 」∠)＿

    作業說明：加入一個刪除按鈕，按下後刪除他在陣列中的資料，並且重新 render 畫面
    https://jsfiddle.net/iamCJ/ndtpurzy/

第四課－熟悉 render function

  • 課程概要：根據資料內容的不同，render function 內可以有各種條件判斷、巢狀結構等等較複雜的邏輯
  • 作業心得：做這個作業沒什麼特別想法，只覺得這樣的寫法真的讓思考變得輕鬆多了 XD

    作業說明：延續上一堂作業，讓 data 加入 category 屬性來做分類，在 render function 設條件判斷，不同分類的事項要有不同顏色
    https://jsfiddle.net/iamCJ/5ba2e3x0/

第五課－熟悉匿名函式

  • 課程概要：
    1. 認識布林值：是資料型態的一種，只有true跟false兩種值
    2. 認識更新陣列值的方法：只要指定陣列的索引，並賦值給它即可
  • 作業心得：覺得這系列的作業跟在系列二寫的時候感覺很不一樣，系列二寫得有點卡，覺得所有功能都尬在一起，有時候看著畫面亂亂的程式碼，腦袋也變得亂亂的了 XD

    作業說明：延續上一堂作業，讓 data 加入 isCompleted 屬性，增加標示已完成的按鈕，按下按鈕後要更新 isCompleted 的狀態，並且更新畫面，讓事項的後面有（已完成）的字樣
    https://jsfiddle.net/iamCJ/kbc96Lq3/

第六課－認識 data model 的優點

  • 課程概要：沒有教新的東西，繼續加強 data model 的練習
  • 作業心得：data model 的設計真的讓取資料變得很簡單耶！記得在系列二時，做這個作業總感覺很麻煩，code 也很不乾脆（？，取資料都要從 DOM 下手，今天就輕鬆多了(ﾉ>ω<)ﾉ

    作業說明：延續上一堂作業，新增一個匯出事項的按鈕，按下按鈕後，要 alert 出所有事項，並依事項的分類（重要程度）有不同的提示（加上米字號）
    https://jsfiddle.net/iamCJ/L8dhzxs0/

第七課－資料序列化

  • 課程概要：
    1. 認識資料序列化（serialization）：即為將 data model 轉為字串，可以用來跟第三方程式溝通、或將資料儲存到資料庫等等
    2. 認識反序列化（deserialization）：則跟序列化相反，是將序列化資料轉換回來，變成程式可用的資料
  • 作業心得：再次讚嘆這個 data model 設計，之前在寫的時候不知道為什麼很容易會有煩躁感（？，現在終於知道原因了，除了是因為之前把資料跟畫面全部寫在一起的關係，另外就是跟我寫在 Vue 的時候差很多，一直很不習慣沒辦法直接取用 data，而現在這個寫法超清爽*｡٩(ˊωˋ*)و✧*｡

    作業說明：延續上一堂作業，新增一個儲存按鈕，按下按鈕後將陣列內容存入 local storage，並且在重整網頁時判斷 local storage 有無資料，如果有則自動載入
    https://jsfiddle.net/iamCJ/huwmsf40/

小結

這系列的學習速度又回來啦！覺得這一系列的寫法比較符合我的習慣，在寫作業的過程跟心情都很順暢 XD。接下來就要進入學習 Vue 的系列啦，之前有自學過，我在想應該可以很快就上手，也期待可以學到新觀念跟新技巧～
