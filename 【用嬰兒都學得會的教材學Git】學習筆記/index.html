【用嬰兒都學得會的教材學Git】學習筆記

發表於 2023-06-05 分類於 Git

前言小廢話

Git 是進入職場必會的技能，要懂得使用它才能夠跟其他人合作專案，如果一個操作失誤，可是有可能讓所有人的心血全部不見、釀成無法挽回的大災難呀 Σ(°△°|||)，因此要好好學習常用的指令，用小練習讓操作熟練，就比較不容易出錯啦～
p.s. 我很不會想名字、標題，想說是同一個人的教學，乾脆就沿用 JS 系列的名稱吧～ ∠( ᐛ 」∠)＿

課程開始

第一課－學會 git 初始化

  • 課程概要：
    1. git init：這個指令會初始化資料夾環境，讓 git 開始追蹤資料夾內所有檔案的任何變化
    2. git status：這個指令可以查看目前的工作狀態
    3. git log：這個指令可以查看歷史編輯紀錄
  • 作業說明：建立新資料夾並用 git 初始化，接著分別輸入git status與git log看看結果如何

第二課－學會 git 基本指令

  • 課程概要：
    1. git add {檔名}：這個指令可以追蹤檔案，將指定的檔案放入暫存區，等待被 commit
    2. git commit -m '描述提交的內容或概要'：這個指令是用來提交放置在暫存區的檔案
  • 作業說明：依順序送出三個 commit，第一個是新增 README.md 後提交，第二個是新增 index.html 後提交，第三個是修改 index.html 內容，並新增 about.html 和 background.html 後提交，最後輸入git log查看結果

第三課－學會 git 版本切換

  • 課程概要：
    1. git checkout {提交代號}：每次 commit 後面都會有一長串英數夾雜的提交代號，輸入git checkout加上代號的指令，可以回朔到該次 commit 的狀態
    2. git checkout master：這個指令則可以回到最新狀態（還是要看你的工作階段是不是在 branch，master 只是最主要的 branch 名稱）
  • 作業說明：使用git checkout指令切回前一個 commit 的版本，並輸入git log查看結果

第四課－學會 git 重置指令

  • 課程概要：
    1. git diff：在將檔案 add 到暫存區之前，使用這個指令可以比對查看檔案有差異、變化的部分
    2. git reset {檔名}：這個指令可以將檔案從暫存區移回草稿區
    3. git checkout -- {檔名}：這個指令可以將檔案的修改內容從草稿區直接取消，回到修改前的狀態
  • 作業說明：這次作業有三個階段，第一是將一個檔案修改內容後 add 到暫存區，第二是使用git reset將檔案移出暫存區，第三是使用git checkout取消修改，每一個階段都輸入git status看看結果如何

第五課－學會連線 github

  • 課程概要：學會將存在本機端的專案上傳到 Github 的 repository
    1. 註冊 Github 帳號後，新增專案儲存庫 Create a new repository
    2. 將專案命名
    3. 專案類型依需求設為公開 public 或私人 private
    4. 建立完成後，專案的網址會長這樣：https://github.com/iam-CJ/Example-of-Git-tutorial
    5. 在本機端專案位置，使用終端機輸入以下指令，將 Git 與專案網址連結：git remote add origin git@github.com:【iam-CJ】/【Example-of-Git-tutorial】.git（將【】部分替換成你的帳戶名稱及專案名稱）
    6. 而每次跟雲端 Git （在這裡是 Github）互動會需要驗證 SSH key，SSH key 設置方法可以參考這篇文章－【Git 教學】手把手 Github SSH 連線設定教學 (Windows/Mac)
    7. 接著建立一個命名為 main 的分支（預設的分支命名是 main 或 master）：git branch -M main
    8. 將本機端的程式碼推送到 Github 上：git push -u origin main
  • 作業說明：將前一課的作業上傳至 Github

    https://github.com/iam-CJ/Example-of-Git-tutorial

第六課－學會 git pull

  • 課程概要：
    1. 將專案clone至本機：有時會遇到使用其他電腦，專案不在該電腦裡，可以到專案的 Github 頁面，點Code按鈕，會顯示 clone 的指令git@github.com:iam-CJ/Example-of-Git-tutorial.git，然後在終端機輸入git clone git@github.com:iam-CJ/Example-of-Git-tutorial.git，就可以將專案複製到電腦中
    2. git pull：用來將專案的最新版本抓下來，同時會將本機端已經存在的 commit 做合併
  • 作業說明：git clone專案到電腦，模擬跨裝置同步專案，(1)首先在其中一個專案資料夾新增檔案，並且 commit 後 push 到 Github，(2)再來到另一個資料夾新增檔案，並且 commit 後 push，這時會出現錯誤訊息，顯示新舊版本發生衝突，(3)因此需要使用git pull抓取專案在 Github 的最新版本，並自動合併剛剛的 commit，最後才能成功將 commit push 出去

    https://github.com/iam-CJ/Example-of-Git-tutorial

第七課－學會處理 git 衝突

  • 課程概要：
    1. 同一份檔案在不同地方被修改並且 push、pull 之後，Git 可能會跳出 commit 衝突的提示，此時打開檔案會看到這樣的內容：
      上半部（HEAD 的部分），是剛提交的 commit 部份，下半部是原本的內容，而最後一行英數夾雜的字串則是從 Github 抓下來的 commit 代號
    2. 解決上述衝突的方法：而解決辦法只要決定衝突內容要刪去、保留還是合併，修改內容並刪去 Git 留下的標示後再次 commit、push 即可
  • 作業說明：接續上一課模擬跨裝置同步專案的狀態，(1)在其中一個專案資料夾修改檔案，並且 commit 後 push 到 Github，(2)到另一個資料夾修改同一份檔案，並且 commit 後 push，這時會出現錯誤訊息，要你 pull 更新專案，(3)pull 後再 push 會發現仍然會有錯誤訊息，檔案的新舊內容有所衝突，打開檔案修改成需要的內容後，(4)再次 commit 並 push 即可

    https://github.com/iam-CJ/Example-of-Git-tutorial

小結

之前有在Learn Git Branching這個網站練習常見的指令及操作，因此這課的概念對我來說是熟悉的，不過我之前都是透過 Fork 這個將指令視覺化的輔助工具來操作 git 的，這次試著用終端機輸入指令練習，滿新鮮但又有點怕，雖然出現的錯誤訊息都淺顯易懂，但還是很怕自己有可能一時腦袋不清楚操作失誤 XD
